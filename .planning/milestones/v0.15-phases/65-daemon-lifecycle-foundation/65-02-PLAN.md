<objective>
Refactor `startServer` in `src/server.js` to support a `managed` mode (D-03), the highest-risk integration of the milestone — sequenced first and verified early. Under `managed:true`, four load-bearing points are gated so the daemon (`run.js`, Plan 03) owns the PID, the signals, and the exit: (1) misconfig throws a discriminated error instead of `process.exit(1)` (server.js:407); (2) `server.listen` gets an `'error'` handler for clean EADDRINUSE (server.js:576, today unhandled → uncaught); (3) it does NOT write its own `server.pid` (server.js:581); (4) it installs NO self `SIGTERM`/`SIGINT`→`process.exit(0)`+`unlinkSync` handlers (server.js:612-618). Managed returns `{ server, stopReconcile }`. The legacy `kodo start` path (`managed:false`, the default) stays byte-identical — zero observable regression (UP-06).

Purpose: This gated refactor unblocks the whole daemon (Plan 03/04) AND the Phase 68 setup mode (no `process.exit(1)` means the first-run chicken-and-egg is escapable). Because it is the riskiest change, the UP-06 regression golden is written and made green FIRST (Task 1) as a safety net, then the refactor lands (Task 2) and must keep that golden green.
Output: `startServer` with an opt-in `managed` mode + optional DI seam, a UP-06 regression golden, and a UP-04 managed-mode unit test.
</objective>

## Interface delivered (for Plan 03 executors)

`startServer(opts = {})` where `opts = { port?, insecure?, managed?, _loadConfig?, _provider? }`.
- Legacy (`managed` falsy): returns `http.Server` (unchanged); writes `server.pid`; `process.exit(1)` on misconfig; installs own SIGTERM/SIGINT cleanup.
- Managed (`managed:true`): returns `{ server, stopReconcile }`; throws `{ code:'KODO_SETUP_REQUIRED' }` on missing secret; rejects `{ code:'EADDRINUSE' }` on port collision; writes NO PID; installs NO signal handlers.
