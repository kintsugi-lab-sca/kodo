**Status:** In planning

## Goal

Verify via **source-level grep** that the API key value **NEVER** leaks to **5 critical sinks**:
1. argv of `execFile`/`spawn` (most dangerous: shell-out)
2. `console.*` (stdout leakage)
3. `logger.*`/NDJSON (log file leakage)
4. `saveConfig`/`config.json` (config file leakage)
5. Overlay snapshot TUI (scrollback leakage)

Complement with runtime UAT: `ps`, `grep ~/.kodo/logs`, `/status` endpoint checks.

## Scope: In

- Static grep test (blindfold against source code patterns)
- Runtime UAT checklist (manual verification after Phase 68 setup mode)
- Perms check: `ls -l ~/.kodo/.env` = `-rw-------` (0600)
- Residual `.env.tmp` check (should not exist after write)
- `process.env[api_key_env]` contains the value (prove caching works)

## Scope: Out

- Vault/secret manager integration (future)
- Rotation policies (future)
- Audit logging of accesses (future)
- Encryption at rest (future; `.env` perms 0600 is v0.15's boundary)

## Success Criteria

1. Γ£à Grep test: no pattern matches `api_key_env`, `api_key`, `PLANE_API_KEY`, `GITHUB_TOKEN` in: `execFile`, `spawn`, `child_process`, `console`, `logger`, `config.json`, overlay snapshot
2. Γ£à Manual UAT: after `writeEnvVar('PLANE_API_KEY', 'secret_key_123')`, run `ps aux | grep kodo` ΓÇö no `secret_key_123` in argv
3. Γ£à Manual UAT: `grep -r 'secret_key_123' ~/.kodo/logs` returns empty (key not in logs)
4. Γ£à Manual UAT: `curl http://localhost:9090/status` response does not contain the key value
5. Γ£à File perms check: `ls -l ~/.kodo/.env` = `-rw-------` after write
6. Γ£à No residual `.env.tmp` file left behind
