## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem (~/.kodo) → proceso | `needsSetup` lee `existsSync(CONFIG_PATH)` + presencia de la key desde `process.env` (poblado por loadEnvFile al import). Entrada de disco no confiable → decisión de arranque. |
| CLI (`kodo up`) → daemon spawn | `runUp` decide spawnear o no el daemon. Un falso negativo de detección engancharía el visor a un proceso muerto (DoS de arranque). |
