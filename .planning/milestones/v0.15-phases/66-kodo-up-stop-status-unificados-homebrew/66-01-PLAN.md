<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operador/shell → CLI (`kodo up`) | argv del operador entra al orquestador |
| CLI → daemon spawn (detached) | `runUp` lanza un proceso hijo vía startDaemon |
| CLI → HTTP `/health` (localhost) | health-wait consume el server local |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-66-01 | Elevation of Privilege | daemon spawn (PATH lookup) | high | mitigate | runUp NO reimplementa spawn; delega en `startDaemon` que usa `process.execPath` + KODO_BIN absoluto + argv array form (lifecycle.js:120-131, cero PATH lookup) |
| T-66-02 | Denial of Service | health-wait loop | medium | mitigate | `waitForHealth` bounded por `timeoutMs` + never-throws; nunca cuelga aunque /health falle siempre |
| T-66-03 | Tampering | daemon persistente al cerrar dashboard | medium | mitigate | runUp no registra signal handlers hacia el daemon; el aislamiento lo da `detached:true` (UP-02 LOCKED) — cerrar el visor no puede tumbar el daemon |
</threat_model>
