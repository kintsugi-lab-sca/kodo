## v0.6 Requirements — Final Status

### Worktree always-on (WT-*)

- [x] **WT-01** — Toda sesión kodo se lanza con `claude --worktree` (full + quick + no-GSD). No depende de labels ni flags opt-in. **Validated** in Phase 18.
- [x] **WT-02** — El path del worktree se deriva determinísticamente del session-id y se persiste en `SessionRecord.worktree_path`. **Validated** in Phase 18.
- [x] **WT-03** — El lock per-repo sigue siendo sobre el repo principal, NO sobre el worktree. Dos sesiones contra el mismo repo siguen coalesciendo. **Validated** in Phase 18.
- [x] **WT-04** — El `stop` hook hace cleanup del worktree (`git worktree remove`) tras release del lock, fail-open si la sesión dejó cambios sin commitear. **Validated** in Phase 19 (con `<wt>.dirty` move-aside).
- [x] **WT-05** — `auto-commit` de la skill opera dentro del worktree (cwd correcto en `stop.js`); `KODO_ROOT` env override sigue funcionando. **Satisfied-by-design** in Phase 19 D-05 — `handleOrchestratorStop` cwd=KODO_ROOT preservado.
- [x] **WT-06** — `kodo gsd verify` lee `VERIFICATION.md` desde el worktree de la sesión. **Validated** in Phase 19 (nullish coalescing fallback aditivo).

### HOOK-01 universal (HOOK-*)

- [x] **HOOK-01** — `buildSessionContext` añade sección "Anti-push-fantasma" a TODAS las sesiones (GSD + no-GSD). Driver: ROMAN-125/126. **Validated** in Phase 20 (ES + EN bloques al final de buildSessionContext + buildGsdContext).
- [x] **HOOK-02** — Golden bytes preservados — tags `[GSD quick]`, `[GSD phase N]`, `[GSD bootstrap]` no cambian shape; bloque HOOK-01 en posición determinista por modo. **Validated** in Phase 20 (append at end → satisfied by construction).
- [x] **HOOK-03** — Test coverage para los 3 modos × `buildSessionContext` cubre que el recordatorio aparece y que el resto del prompt no muta. **Validated** in Phase 20 (+14 tests; matrix 4 modos × HOOK-01/02/03).

### Skill sync (SKILL-*)

- [x] **SKILL-01** — CLI `kodo skill sync` empuja `<repo>/.claude/skills/` → `~/.claude/skills/`. Diff-aware SHA-256; NO borra archivos foráneos salvo `--prune` explícito. **Validated** in Phase 21.
- [x] **SKILL-02** — `kodo orchestrator` detecta drift y sincroniza automáticamente con evento `skill.sync.auto` en NDJSON. **Validated** in Phase 21 (hook before `cmux.listWorkspaces`).
- [x] **SKILL-03** — Auto-sync NO rompe Constraint cwd=repo Phase 999.1. **Validated** in Phase 21 (process.cwd() intacto; skill local del repo sigue ganando).
- [x] **SKILL-04** — Exit codes deterministas: 0 sync ok / 0 no-op / 1 fs error / 2 fuera de repo. **Validated** in Phase 21 (4 spawnSync tests + canonical stderr).
