| Boundary | Description |
|----------|-------------|
| CLI operator → doctor execute | `--fix` is the single explicit opt-in that triggers destructive filesystem/git mutation |
| dry-run preview → operator trust | The operator runs `--fix` in production based on the dry-run report; an inaccurate preview causes unexpected deletion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-09 | Elevation of Privilege | `--fix` accidental invocation | mitigate | Dry-run-by-default (no flags = read-only) imposes the 2-step ritual; `--fix` is the only mutation opt-in (D-03/D-07). No silent mutation path. Exit-code semantics let CI gate on `1` before any human runs `--fix`. |
| T-41-10 | Spoofing | inaccurate dry-run preview | mitigate | D-08: the human render shows the EXACT per-item action (remove/prune/move/steal/keep/unlink), so the operator sees precisely what `--fix` will do. Human UAT step #2/#5 verifies preview matches actual mutation. |

| T-41-12 | Tampering | live-resource deletion | mitigate | Per-action liveness re-check (Plan 02 D-06/D-14) + human UAT step #4/#5 confirm live worktree/lock/log are protected. |
