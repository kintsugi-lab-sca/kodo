## Fuente única de alive + guard Enter (TUI-13/14)

Base no controvertida (sin opciones): quitar el override legacy de `alive` en `/status` (server.js:381) para que mande el `reconcileTick` (host --json autoritativo). Sub-decisión sobre el guard de Enter:

| Option | Description | Selected |
|--------|-------------|----------|
| Fail-open: permitir focus | Mantener guard actual (solo bloquea con alive===false). undefined → permite attach; el fallo real lo captura FOCUS_ERR_*. | ✓ |
| Fail-closed: bloquear hasta confirmar vivo | Guard 'solo permite con alive===true'. Más seguro pero bloquea momentos válidos durante arranque/primer tick. | |

**User's choice:** Fail-open: permitir focus (recomendado).
**Notes:** Coherente con la degradación elegante del milestone; no se castiga al operador por un tick aún no llegado. Nota técnica: displayStatus (server.js:199-200) también consume `alive` → debe leer el mismo valor autoritativo.
