    Create `src/triggers/` directory and `src/triggers/dispatcher.js`.

    Extract the logic currently in `handleTriggerState()` (server.js lines 82-141) into a pure `dispatchTrigger(event, opts)` function:

    ```javascript
    /**
     * @param {import('../interface.js').TriggerEvent} event
     * @param {{ model?: string|null, flags?: string[], force?: boolean }} [opts]
     * @returns {Promise<{ action: 'launched'|'ignored'|'already_active'|'stale_relaunch', session?: object }>}
     */
    export async function dispatchTrigger(event, opts = {}) { ... }
    ```

    Implementation details:
    1. Get provider via `getProvider()` from registry (do NOT call `initRegistry()` — caller is responsible)
    2. Resolve task: `const task = await provider.getTask(event.taskRef)`
    3. Check kodo labels: `parseKodoLabels(task.labels.map(n => ({ name: n })))` — skip if `opts.force === true`
    4. Session-already-active guard: check `listSessions()` for matching `task_id === task.id`, verify workspace via `cmux.listWorkspaces()`. If workspace alive → return `{ action: 'already_active' }`. If workspace gone → `removeSession(task.id)`, continue.
    5. Launch: `const session = await launchWorkItem(event.taskRef, { model: opts.model ?? kodoConfig.model, flags: [...(opts.flags || []), ...kodoConfig.flags] })`
    6. Return `{ action: 'launched', session }` (or 'stale_relaunch' if cleaned stale first)

    For tests: mock provider, manager, state, cmux, and labels modules. Use `node:test` mock facilities. Test file goes in `test/dispatcher.test.js`. Structure tests as `describe('dispatchTrigger', () => { ... })` with setup/teardown for mocks.

    IMPORTANT: The `parseKodoLabels` function in `src/labels.js` expects an array of objects with `name` property (label objects), but `TaskItem.labels` is `string[]`. The dispatcher must bridge this: `task.labels.map(name => ({ name }))`.

  <action>
    Create `src/triggers/webhook.js` exporting `handleWebhookRequest()`:

    ```javascript
    /**
     * Pure webhook handler — receives data, returns data. No HTTP req/res.
     * @param {string} rawBody - Raw HTTP body string
     * @param {object} headers - HTTP headers object
     * @param {import('../interface.js').TaskProvider} provider - Active provider adapter
     * @returns {Promise<{ status: number, body: object }>}
     */
    export async function handleWebhookRequest(rawBody, headers, provider) { ... }
    ```

    Implementation flow:
    1. `if (!provider.verifySignature(rawBody, headers))` → return 401
    2. Try `JSON.parse(rawBody)` → catch → return 400
    3. `const triggerEvent = provider.parseTriggerEvent(payload)` → if null → return 200 with `ignored: true`
    4. Fire-and-forget: `dispatchTrigger(triggerEvent).catch(err => console.error(...))`
    5. Return `{ status: 200, body: { ok: true } }`

    Key design decisions:
    - The function does NOT await dispatchTrigger — webhooks must respond fast (preserves current server.js behavior)
    - The function has ZERO provider-specific logic — all delegation goes through the provider adapter interface
    - Import `dispatchTrigger` from `./dispatcher.js`

    For tests: mock the provider object (verifySignature, parseTriggerEvent) and mock dispatchTrigger. Use `node:test` mock.module for the dispatcher import. Test file in `test/webhook.test.js`.
