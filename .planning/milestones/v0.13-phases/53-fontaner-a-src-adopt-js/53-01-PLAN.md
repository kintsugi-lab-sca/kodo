<objective>
Upgrade the single state-write chokepoint `saveState` (`src/session/state.js:241-242`) from a plain `writeFileSync` to an atomic tmp+rename, satisfying BIDIR-05's durability requirement (D-05). This is a surgical one-function change: every state writer (`addSession`/`updateSession`/`removeSession`) inherits atomic durability transparently. Adoption's `PERSIST_FAILED` LOUD path (Plan 53-02) depends on this write being durable.

Purpose: `state.json` integrity is the central correctness invariant of kodo. A torn write on crash would corrupt the model the entire system reads. tmp+rename makes partial writes structurally impossible.
Output: `saveState` writes via `${STATE_PATH}.tmp` + `renameSync`; a regression test proves atomicity AND that the `.bak` migration snapshot path is unaffected.
</objective>
