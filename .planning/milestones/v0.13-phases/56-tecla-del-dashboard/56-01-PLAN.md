## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cmux output → dashboard | `cwd`/`workspaceRef`/`sessionId` values originate in cmux `resume_binding`, already string-validated by `normalizeSurface` (src/host/cmux.js:46-67), flow into runAdopt's argv |
| operator config → reverse-lookup | `loadProjects()` map is operator-controlled; `cwd` is compared against it as a string, never used to build filesystem paths |
| dashboard → child process | runAdopt spawns `kodo adopt` via execFile (no shell) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01 | Tampering / EoP | runAdopt argv construction (adopt.js) | mitigate | execFile (no shell) with a LITERAL argv array — shell metacharacters in cwd/sessionId are inert; each value is preceded by its explicit `--flag` so a value beginning with `-` is consumed as the flag's argument, not a new flag. Test 1 asserts the literal argv shape. |
| T-56-02 | EoP | kodo binary resolution (adopt.js) | mitigate | Binary is `process.execPath` (node) with `kodoBin` as an absolute path argv[0] — zero PATH lookup (mirror of polling.js EoP mitigation). No `'kodo'` in PATH. |
| T-56-03 | Tampering (path traversal) | resolveProjectId (select.js) | accept | `cwd` is only string-compared against the operator's `loadProjects()` map; never passed to readFileSync/path-join for FS access. No traversal surface. Normalize with path.normalize only (no realpathSync — keeps the helper pure). |
| T-56-04 | DoS (UX) | runAdopt never-throws contract | mitigate | All failures (ENOENT/non-zero/spawn/sync-throw) collapse to a resolved {ok:false}; the promise never rejects, so no throw can reach React. Test 4 asserts no-reject on sync throw. |
| T-56-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — zero external dependencies added (RESEARCH §Standard Stack). No legitimacy gate required. |
</threat_model>
