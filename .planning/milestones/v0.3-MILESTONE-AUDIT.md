## Cross-Phase Integration

**Flow 1: GSD label → session launch (GSD-01 through GSD-04, GSD-08, GSD-09, GSD-10)** — WIRED end-to-end:
dispatcher recognizes `kodo:gsd` → acquires per-repo lock → resolvePhase matches title → launchWorkItem spawns session → buildGsdContext injects brief/commands → stop hook releases lock + emits `kodo gsd verify` nudge.

**Flow 2: Orchestrator verify gate (GSD-05, GSD-06, GSD-07)** — WIRED:
`## Sesiones GSD` section in prompt.md → `[GSD phase N]` tag in buildContextSummary → `kodo gsd verify <session-id>` CLI → runGsdVerify reads VERIFICATION.md → computes verdict → posts Plane comment → transitions state only on pass → emits `orchestrator.review` event.

**Flow 3: Logs CLI (LOG-05, LOG-06, LOG-07, LOG-11)** — WIRED:
`kodo logs [session-id]` → reader.js → follow.js (watchFile 200ms) + level/component/event-type filters + session-lookup.js for --session-of (two-step: state.json → head-line scan).

**Flow 4: LOG-12 guard** — VERIFIED: `kodo check` has zero logger.js imports; `test/check-isolation.test.js` enforces 4 guards.

## Cross-Phase Issues (4 findings)

### INT-HIGH-01 — plane.api.call.failed bypasses EVENTS catalog
**File:** `src/gsd/verify.js:185, 196, 213`
**Affects:** LOG-09 (taxonomy completeness), GSD-06 collateral
**Issue:** Tres paths de error de Plane side-effects emiten `plane.api.call.failed` como string crudo en lugar de usar un helper del catálogo `EVENTS` en `src/logger-events.js`. Bypassa el typo-guard y deja los errores invisibles para `kodo logs --event-type plane.api.call`.
**Cross-ref:** 10-REVIEW.md WR-03
**Fix:** Add `PLANE_API_CALL_FAILED: 'plane.api.call.failed'` to EVENTS + `planeApiCallFailed()` helper; replace raw strings in verify.js.

### INT-MED-01 — dispatcher raw strings
**File:** `src/triggers/dispatcher.js:169, 200`
**Affects:** LOG-09
**Issue:** Usa literales `'gsd.phase.resolved'` y `'gsd.bootstrap'` en vez de `EVENTS.GSD_PHASE_RESOLVED` / `EVENTS.GSD_BOOTSTRAP`. Runtime-compatible pero bypassa typo-guard. Deferido explícitamente en Phase 9.
**Fix:** Dynamic-import EVENTS + substitute.

### INT-LOW-01 — state.transition never emitted in production
**File:** `src/session/manager.js`
**Affects:** LOG-09
**Issue:** `markSessionStatus()` exportado pero sin callsites de producción. Phase 7 D-06 "NO rewiring forzado" diferió el wiring.
**Fix (futuro):** Wire `markSessionStatus` en las transiciones del dispatcher y stop hook.

### INT-LOW-02 — doc mismatch plane_task_id vs task_id
**File:** `.planning/phases/07-kodo-logs-cli-event-taxonomy/07-01-SUMMARY.md`
**Affects:** LOG-10 (documentation)

**Fix:** Actualizar SUMMARY doc.
