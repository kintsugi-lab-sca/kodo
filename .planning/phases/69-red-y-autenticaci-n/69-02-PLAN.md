<objective>
Wire the Plan 01 auth primitives into the running `node:http` server: default-deny bearer middleware on every non-webhook route, safe-by-default bind host, a 1 MB pre-auth body limit, neutral 500 error bodies, and token passing for the embedded web dashboard. This is the core of NET-01..04 plus the web-dashboard half of NET-02 (D-05).

Purpose: Close the external network surface (causa raíz T3) — the server stops listening on all interfaces by default and the non-webhook rail rejects unauthenticated requests, without leaking data, HTML, or internal error text to an attacker. All changes are additive to existing 200 responses (401/413 are new per-route codes, not shape changes — invariant v0.9). Zero new endpoints (invariant since v0.10).

Output: hardened `src/server.js` + integration tests over a real ephemeral-port server.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN/external client → HTTP server | Untrusted requests cross here; the bind + bearer + body-limit + error-hygiene controls all live at this boundary |
| browser navigation → HTML dashboard | The dashboard shell is served via a query-param token (accepted tradeoff for local use); the primary API rail uses the Authorization header |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-69-01 | Spoofing / Information Disclosure | server.listen bind host | high | mitigate | Bind `127.0.0.1` by default (`config.server.bind ?? '127.0.0.1'`) at both listen sites; explicit opt-in to expose (NET-01) |
| T-69-02 | Elevation of Privilege / Tampering | non-webhook route middleware | high | mitigate | Default-deny bearer guard on every route except GET /health and POST /webhook; timing-safe compare; neutral 401 (NET-02, D-04) |
| T-69-03 | Denial of Service | readBody body accumulation | high | mitigate | 1 MB cap → 413 emitted PRE-auth/PRE-HMAC; `req.destroy()` on overflow (NET-03, D-06) |
| T-69-05 | Information Disclosure | 500 error response bodies | medium | mitigate | Neutral `{error:'internal error'}` body; thrown message logged only (NET-04) |
| T-69-07 | Information Disclosure | ?token= query param on HTML routes | low | accept | Query-param token used ONLY for `/` and `/dashboard`; documented tradeoff for the local dashboard; API rail uses the Authorization header (D-05) |
| T-69-SC | Tampering | npm/pip/cargo installs | high | accept | No package installs; `node:http` + `node:crypto` + internal `src/server/auth.js` only — zero-new-dependency invariant |
</threat_model>
