  <action>
    Create `src/server/auth.js` (@ts-check, ESM) exporting five pure/DI helpers plus one constant. Use `node:crypto` `timingSafeEqual` and `randomBytes`, and import `writeEnvVar` from `../config.js`.

    parseBearer(headerValue): return the token after a case-insensitive `Bearer ` prefix (trimmed), else null. Non-string input returns null. Match per RFC 6750.

    timingSafeTokenEqual(provided, expected): return false if either is falsy; build Buffers of both; return `a.length === b.length && timingSafeEqual(a, b)` inside a try/catch that returns false on throw. This is the exact shape of the Plane HMAC compare (D-03) — the length guard prevents timingSafeEqual from throwing on unequal-length inputs.

    isOpenRoute(method, pathname): return true ONLY for (`GET` + `/health`) or (`POST` + `/webhook`). Everything else is closed (default-deny allowlist of the two OPEN routes, per D-04). Do NOT enumerate closed routes.

    getOrCreateApiToken({ env = process.env, writeEnvVarFn = writeEnvVar } = {}): if `env.KODO_API_TOKEN` is a non-empty string, return it unchanged (idempotent, no write). Otherwise generate `randomBytes(32).toString('hex')` (64 hex chars — parser-safe; NEVER base64 standard, which contains `+`/`/`/`=` and would make writeEnvVar throw — Pitfall 1). Call `writeEnvVarFn('KODO_API_TOKEN', token)`; if it returns false, throw an Error with `code: 'KODO_TOKEN_WRITE_FAILED'` (do not start with auth silently disabled — D-02). On success set `env.KODO_API_TOKEN = token` (in-process cache, since loadEnvFile is load-no-override) and `console.log('[kodo] auth token: ENABLED')` — log the word ENABLED only, NEVER the value (PERSIST-04). Return the token.

    Export `MAX_BODY_BYTES = 1024 * 1024` (1 MB — the pre-auth body cap constant Plan 02's readBody consumes for NET-03).

    Create `test/server/auth.test.js` covering every bullet in `<behavior>`. Use `node:test` + `node:assert/strict`. For getOrCreateApiToken inject a fake `env` object and a fake `writeEnvVarFn` (capturing key/value, returning true or false per case) — no real `~/.kodo/` write, no HOME manipulation needed for these units. Capture console.log via a temporary override to assert the token value never appears in output.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| token storage → runtime | The bearer secret is generated once and persisted to `~/.kodo/.env` (0600); it must never leak to logs, argv, or `/status` |
| token compare | The provided token (attacker-controlled) is compared against the real token; the comparison must not leak byte-position via timing |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-69-04 | Information Disclosure | timingSafeTokenEqual (token compare) | medium | mitigate | `crypto.timingSafeEqual` with length guard (never `===`); mirrors the Plane HMAC pattern (D-03) |
| T-69-08 | Information Disclosure | getOrCreateApiToken (token gen/persist/log) | medium | mitigate | Token generated with `randomBytes(32)` (CSPRNG, 256-bit); persisted via `writeEnvVar` (0600 pre-rename); log emits only the word ENABLED, never the value (PERSIST-04) |
| T-69-SC | Tampering | npm/pip/cargo installs | high | accept | No package installs in this plan; cross-milestone zero-new-dependency invariant — only `node:crypto` built-in + internal `writeEnvVar` |
</threat_model>
