<objective>
Make the Ink TUI dashboard authenticate every request with the bearer token and surface a 401 as a clear, visible degradation state. Per D-07 (one token read, one authed fetch wrapper, zero client.js signature changes) and D-08 (never a silent empty screen; a distinct "no autorizado" banner).

Purpose: The Ink dashboard is one of only two authenticated consumers of the hardened API rail (the other is the embedded web dashboard, handled in Plan 02). It must attach the token the server now requires and, when the token is missing/revoked, degrade visibly with a remediation hint rather than crashing or blanking — following the established never-throws / visible-degradation pattern.

Output: authed dashboard client wiring + a dedicated 401 UI state, both unit-tested with injected fetch/harness (no real server needed).
</objective>
