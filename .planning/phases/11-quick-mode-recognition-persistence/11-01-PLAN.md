Texto exacto a añadir:

```javascript

/**
 * Returns the GSD execution mode encoded in a persisted Session record.
 * Centralized here (paired with getGsdMode) so hooks, orchestrator and tests
 * share one definition of "what mode is this session?".
 *
 *   gsd:true, gsd_mode:'quick' → 'quick'
 *   gsd:true, gsd_mode:'full'  → 'full'
 *   gsd:true, no gsd_mode      → 'full'   // legacy preservation (Phase 11 D-08):
 *                                          //   sesiones pre-v0.4 con `gsd:true`
 *                                          //   eran siempre full por contrato.
 *   gsd:false / missing        → null     // non-GSD session
 *
 * El helper vive en labels.js (no en session/state.js) porque la regla
 * "legacy gsd:true == full" es semánticamente parte de la taxonomía de
 * labels: define qué label histórica equivale a qué modo (matiz Phase 11
 * <specifics>).
 *
 * Defensivo: nunca lanza para session null/undefined/sin campos.
 *
 * @param {import('./session/state.js').Session | null | undefined} session
 * @returns {'full'|'quick'|null}
 */
export function getSessionMode(session) {
  if (!session?.gsd) return null;
  return session.gsd_mode || 'full';
}
```

Notas de implementación:
- El operador `?.` defiende `session=null/undefined`. Sin él, `null.gsd` lanzaría.
- `session.gsd_mode || 'full'` aplica D-08 (ausente → full). NO usar `??` aquí: si en el futuro alguien escribiese `gsd_mode: ''` (string vacío), queremos que también colapse a `'full'`, no propagar el vacío.
- NO importar `Session` en runtime — el `@param` con `import('./session/state.js').Session` es solo para `// @ts-check` typing, no genera código.
- Mantener el orden de export estable: `parseKodoLabels` (línea 12), `getGsdMode` (línea 53), `getSessionMode` (nuevo, al final). Phase 12 hooks importarán los tres en una sola línea: `import { parseKodoLabels, getGsdMode, getSessionMode } from '../labels.js';`.

Por qué se hace así (D-03/D-09 razonamiento):
- Simetría con `getGsdMode(flags)`: ambos helpers viven juntos, ambos centralizan una regla de derivación, ambos son consumidos downstream sin replicación.
- D-10 explícito: aunque los consumers viven en Phase 12, el helper se introduce ahora porque es testeable aislado y porque la firma `Session` ya está estable en state.js (WIP merged en este phase).
