## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kodo local → external task manager (POST) | Title/description cross from the local machine into a third-party-hosted task manager. Untrusted-to-leak surface: absolute paths, home-dir disclosure, transcript bodies. |
| createTask POST → local state write | Between a successful remote POST and the local seed-row write lies an orphan window: a task created with no local record. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-03 | Information Disclosure | sanitizeAdoptionData (title/description → external POST) | mitigate | Redact homedir → `~` and strip absolute paths before the createTask POST (BIDIR-08/D-06). Core backstop — applies even if the Phase 57 orchestrator consumer fails to sanitize. |
| T-53-04 | Information Disclosure | transcript exfiltration into provider | mitigate | Structural: sanitizeAdoptionData has NO transcript parameter — it cannot forward a transcript body (defense by construction, not a filter). |
| T-53-05 | Tampering / Repudiation | provider orphan (task created, no local row) | mitigate | PERSIST_FAILED LOUD discriminant carrying task_id + task_url + re-run hint (BIDIR-05/D-03). Never thrown, never silently swallowed; recoverable by idempotent re-run (kodo never deletes). |
| T-53-06 | Elevation / DoS (session storm) | adopted task re-dispatched (recursion) | accept | Already mitigated Phase 52: `kodo:adopted` marker + dispatcher.js:82 cut (pre-lock, `--force`-proof). Adoption inherits it; adopt.js adds no new control. |
| T-53-07 | Tampering | concurrent double-adopt (same operator/daemon) | accept | Residual is negligible (D-04); a true concurrent double-adopt is visible via the `kodo:adopted` marker and recoverable by idempotent re-run. | T-53-SC | Tampering | npm/pip/cargo installs | accept | No package installs — zero new runtime dependencies (node:fs/path/os builtins + existing kodo exports). No registry interaction, no slopcheck surface (RESEARCH §Package Legitimacy Audit: Not Applicable). |
