# Phase 44 — UI Design Contract

> Visual and interaction contract for **Phase 44: Overlay de plan GSD + pulido de dashboard**.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Surface type: TERMINAL TUI (ink), NOT web.** This phase ships **zero CSS/HTML/Tailwind/hex/rem/px**.
> The "design system" is ink primitives (`<Box>` flex + `<Text color>` string color names) over monospace
> terminal cells. Where the standard UI-SPEC template asks for web tokens (hex, rem, fonts, breakpoints),
> this contract substitutes terminal equivalents and marks inapplicable web dimensions
> **"N/A — terminal TUI"** rather than inventing values.

**fixed column widths (in character cells)** and **single-cell padding/margins via ink `<Box>` props**
(`marginTop:1`, `marginBottom:1`, `width:N`). The web 8-point scale does not apply and MUST NOT be invented.

## Typography

**N/A — terminal TUI.** No font sizes (px), no font weights beyond the terminal's bold attribute,
no line-height ratios. The web typography table does not apply. Terminal-equivalent "type" controls:

| Web role | Terminal equivalent | Phase 44 usage |
|----------|---------------------|----------------|
| Font size | fixed monospace cell (operator's terminal) | not chosen by kodo |
| Font weight | ink `bold:true` (single bold attribute) | overlay title `bold:true`; selected row `bold:selected` (existing) |
| Dim / de-emphasis | ink `dimColor:true` | column headers, informational empty-state copy, footer hint, degraded `—`/`?` cells (existing) |
| Line height | one terminal row per `lines[]` entry | overlay body renders one `<Text>` per array line |
| Heading vs body | overlay **title** (cyan bold) vs **body** (default/dim) | `plan` overlay reuses this exact two-level hierarchy |

**Two-weight discipline (terminal-equivalent of the "2 weights" rule):** the TUI uses exactly two
emphasis levels — **bold** (titles, selected row) and **dim** (headers, footnotes, empty states) —
over the normal weight. No third emphasis is introduced.

## Color

**N/A — web hex.** Substitute ink **string color names**. The palette is **already LOCKED** in
`format.js` (`statusColor`, `STATE_BADGES`) and the overlay renderer (`renderOverlay`). Phase 44
introduces **zero new colors** — D-09 explicitly reuses the existing zombie red.

### Color contract (string names only; the "60/30/10" maps to dominant-neutral / structural-dim / reserved-semantic)

| Role | ink value | Usage (this phase) |
|------|-----------|--------------------|
| Dominant (≈60%) — neutral plan/table text | terminal default (no `color`) | Plan body lines, most table cells. The bulk of the surface is uncolored monospace text. |
| Secondary (≈30%) — structural de-emphasis | `dimColor:true` | Column headers, footer hint (`↑↓ scroll · Esc close`), informational empty states (`no-phase`, `no-plan`), degraded markers. |
| Accent (≈10%) — reserved semantic | `cyan` | **Reserved for: overlay title** (`plan · <task_ref>`, bold cyan — mirror of `comments`/`logs`). Also the existing `review` status. NOT used for plan body text. |
| Semantic — error | `red` | Plan overlay `error` status copy (`error reading plan`). **Also TUI-19 zombie per-row mark** (`▶ running (zombie)` in the `state` cell), sourced from `statusColor(status,alive,state)` which already returns `{color:'red'}` for `running + !alive`. |
| Semantic — other states (unchanged, inherited) | `green` running·alive · `yellow` idle · `cyan` needs-input/review · `red` dead/zombie · `magenta` error-status | Existing `STATE_BADGES`/`statusColor`; Phase 44 does not alter these. |

**Accent reserved for:** the overlay **title line only** (cyan bold). Never applied to plan body content,
never to "all interactive elements" (the TUI has no hover/focus color states — interaction is keypress-driven).

**Zombie red reservation (D-09):** red on a `state` cell means specifically `running + alive===false`
(zombie). The mark is **additive** — the header zombie counter (`countsLabel`) is kept, not replaced.
No new red literal: the color is read from the already-v3-aware `statusColor`.

## Copywriting Contract

Copy is the **primary design surface** of this phase. The contract is **"distinct copy per case + honest"**
(D-07). Exact wording is Claude's Discretion (planner/executor refines), but each case MUST be
lexically distinct and non-alarming. Constants live as `OVERLAY_PLAN_*` in `App.js`, lexical mirror of
the existing `OVERLAY_COMMENTS_*` / `OVERLAY_LOGS_*`.

| Element | Copy (recommended; wording is discretion, distinctness is contract) | Color treatment |
|---------|---------------------------------------------------------------------|-----------------|
| Primary interaction (key, not a CTA button) | Press **`p`** (mnemonic "plan") on a selected row to open the plan overlay — sits next to `c` (comments) / `l` (logs). | n/a (keypress) |
| Overlay title | `plan · <task_ref>` (mirror of `comments · <ref>` / `logs · <ref>`) | `cyan`, `bold` |
| Empty state — non-GSD / no phase resolved | `OVERLAY_PLAN_NO_PHASE` → e.g. `not a GSD session / no phase resolved` | `dimColor` (informational, NOT red — this case is **common** because the row lacks `task.title`, so it must read as ordinary, not as a failure) |
| Empty state — phase resolved, no `PLAN.md` yet | `OVERLAY_PLAN_NO_PLAN` → e.g. `phase has no PLAN.md yet` | `dimColor` (informational, not red) |
| Error state — filesystem read failure | `OVERLAY_PLAN_ERROR` → e.g. `error reading plan` | `red` (mirrors existing `error`/`not-found` red treatment) |
| Multiple PLAN.md (NOT an empty state — D-06) | Concatenated in one flat `lines[]`, each file prefixed by a header rule `── <filename> ──` + blank line, sorted ascending by filename. A single unreadable file degrades to `── <filename> (unreadable) ──` without aborting the rest. | default text; separator may be `dimColor` (discretion) |
| Footer hint (reused verbatim) | `↑↓ scroll · Esc close` | `dimColor` |
| Destructive actions | **None.** The overlay is strictly **read-only** (D-11). No write, no delete, no confirmation flow. The only read-write TUI surface remains the v0.10 dismiss (untouched here). | n/a |

**Honesty rule (load-bearing, PLAN-02):** the operator must distinguish at a glance —
"this task isn't GSD" vs "this phase has no PLAN.md yet" vs "there was a read error" — **three distinct
messages, never one generic 'nothing to show'.** The `no-phase` and `no-plan` cases are dim/non-alarming;
only genuine FS errors get red. This mirrors how the `c` overlay separates `unsupported`/`empty`/`not-found`/`error`.

## Interaction & State Contract (terminal-TUI addendum)

> The web template has no section for keypress/overlay interaction. This addendum is the load-bearing
> interaction contract for an ink TUI and is consumed by planner/executor/checker as the interaction source of truth.

| Concern | Contract | Source |
|---------|----------|--------|
| Open | `p` in `mode==='list'` on the selected row (`filtered[sel.index]`). Fourth overlay consumer of `mode:'overlay'`, alongside `c`/`l`. | D-01, D-02 |
| Snapshot | **Frozen** at open via `setOverlaySnapshot({ kind:'plan', taskRef, status, lines })`. The live poll keeps refreshing the table underneath; the overlay content does not change until reopened. | D-02 |
| Scroll | `scrollOffset` over `lines[]`, slice height `OVERLAY_VIEWPORT=18`, clamp `max = lines.length - 18`. `↑/↓` reuse the existing overlay scroll branch unchanged. | D-02 |
| Close | `Esc` → `mode:'list'`; `selectedTaskId` untouched → **cursor preserved by `task_id`**. The Esc branch increments `overlayReqRef` to invalidate any *other* in-flight `c`/`l` open. | D-02 |
| Async guard divergence (IMPORTANT) | The `plan` read is **synchronous** (`readPlan` is a sync DI helper) → **no `await` window → no stale-reopen race → the `p` open does NOT need an `overlayReqRef` capture.** This is a deliberate divergence from the async `c`/`l` mirror. Do **not** cargo-cult a `reqId` capture onto the sync `p` open (it would be dead/misleading code). | RESEARCH Pattern 1, Pitfall 1 |
| Never-throws | ALL filesystem reads (dir resolution, glob, `readFile`) collapse to a discrete `status` (`ok`/`no-phase`/`no-plan`/`error`); no FS error reaches React. The handler never bare-`await`s. | D-05, STATE invariant "TUI nunca crashea" |
| Selection identity | Overlay opens over the row revalidated by `task_id`; filters use `String.includes` (anti-ReDoS, D-13); filename/phase matching uses `String.startsWith`/`endsWith`, never `new RegExp`. | D-13 |

### TUI-18 — column-hide derivation (structural, filter-insensitive)

`anyGsd = rows.some(r => r.phase_id != null)` derived in `select.js` (`deriveAnyGsd`) over the
**unfiltered `sorted`/`sessions`** set — **NOT** over `filtered`. The `phase/mode` column must be
**structural**: it must not flicker as the operator types a `/` filter (e.g. `s:done` momentarily
emptying GSD rows must NOT vanish the column). When `anyGsd===false`, the column `<Box>` (header + every
data cell) is not emitted; it reappears automatically when a row with `phase_id` enters.
