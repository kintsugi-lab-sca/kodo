## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → readPlan | `.planning/phases/**` content (operator's own versioned files) crosses into the helper; path components come from kodo's own `state.json` (`phase_id`, `worktree_path`/`project_path`), not from live operator input |
| plan content → ink `<Text>` render | PLAN.md text is rendered line-by-line; the only render path is ink `<Text>` (escapes; no raw ANSI passthrough) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-01 | Tampering | path traversal via phase_id / dir name in `plan.js` | mitigate | `phase_id` originates from kodo's own `state.json`; paths composed with `node:path.join` under a fixed `.planning/phases/` root; only directory entries already present are read (`readdir` then `find`), never an attacker-supplied path segment — no `..` concatenation from untrusted input |
| T-44-02 | Denial of Service | ReDoS via regex on filenames | mitigate | `String.startsWith`/`endsWith` only; never `new RegExp` (D-13). Asserted by the anti-ReDoS test case in test/dashboard-plan.test.js |
| T-44-03 | Denial of Service | uncaught throw from fs reaching React (crash) | mitigate | never-throws contract (D-05): every readdir/readFile/dir-resolution wrapped; failures collapse to status 'error'/'no-plan'/'no-phase'. The `p` handler never bare-awaits. Guarded by "TUI nunca crashea" invariant |
| T-44-04 | Tampering/Spoofing | ANSI/color injection via plan content | accept | plan content is the operator's own versioned text, rendered through ink `<Text>` (ink escapes; no raw ANSI passthrough). Low trust concern, displayed verbatim line-by-line |
| T-44-05 | Denial of Service | unbounded read of a huge PLAN.md | accept | render is bounded by OVERLAY_VIEWPORT (18-line slice); plans are small text files. A pathological multi-MB plan is held in memory but not a realistic local-TUI threat |
| T-44-SC | Tampering | npm/pip/cargo installs | accept | zero new packages this phase (RESEARCH: "Package Legitimacy Audit — Not applicable"); no install task, no slopcheck checkpoint required |

No high-severity network/auth threats apply: zero new endpoints (D-10), read-only (D-11), no HTTP session, local filesystem only.
