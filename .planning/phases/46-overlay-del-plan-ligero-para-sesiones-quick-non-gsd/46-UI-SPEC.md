## Color

**N/A вЂ” web hex.** Substitute ink **string color names**. Phase 46 introduces **zero new colors** and
reuses the existing, already-LOCKED overlay palette. The only color decision is which existing treatment
the new state gets:

| Role | ink value | Phase 46 usage |
|------|-----------|----------------|
| Dominant (в‰€60%) вЂ” neutral plan text | terminal default (no `color`) | light-plan body lines when `status==='ok'` (inherited plain render) |
| Secondary (в‰€30%) вЂ” structural de-emphasis | `dimColor:true` | **`no-light-plan` copy line (NEW)** вЂ” same dim treatment as `no-phase`/`no-plan`. Also footer hint, headers (inherited). |
| Accent (в‰€10%) вЂ” reserved semantic | `cyan` | overlay title only (inherited, unchanged) |
| Semantic вЂ” error | `red` | `OVERLAY_PLAN_ERROR` for the light-plan `error` case (EACCES/non-ENOENT) вЂ” **reuses the existing red error constant**, no new literal |

**Critical color decision (the one substantive call this phase makes):** the `no-light-plan` state is
**dim, NOT red.** A quick/non-GSD session that simply hasn't written its lightweight plan yet is
**normal and expected** (latest-wins, the session may not have run the instruction yet) вЂ” it is an
*informative* empty state, exactly like `no-phase`/`no-plan`, and must NOT read as a failure. Red is
reserved for genuine filesystem errors (`OVERLAY_PLAN_ERROR`, the EACCES/non-ENOENT path, D-05).

**Accent reserved for:** the overlay title line only (inherited). The new state never uses accent.

## Copywriting Contract

Copy is the **only design surface this phase touches.** The contract is **"distinct copy per case +
honest"** (inherited Phase 44 D-07) extended to a **fourth** case. Constant lives as `OVERLAY_PLAN_NO_LIGHT`
in `App.js`, lexical sibling of the existing `OVERLAY_PLAN_*`.

### The four plan-overlay empty/error copies вЂ” full set after this phase (distinctness is the contract)

| Status | Constant | Literal copy | Color | Meaning the operator must read |
|--------|----------|--------------|-------|--------------------------------|
| `no-phase` | `OVERLAY_PLAN_NO_PHASE` | `not a GSD session / no phase resolved` | `dimColor` | inherited вЂ” row has no resolvable GSD phase **and** no usable `task_id` (terminal case, D-06) |
| `no-plan` | `OVERLAY_PLAN_NO_PLAN` | `phase has no PLAN.md yet` | `dimColor` | inherited вЂ” GSD phase resolved but its `PLAN.md` is absent (GSD-only, never reached by the light fallback) |
| **`no-light-plan`** | **`OVERLAY_PLAN_NO_LIGHT`** | **`session has not written a plan yet`** | **`dimColor`** | **NEW вЂ” quick/non-GSD session, artifact `~/.kodo/plans/<task_id>.md` absent (ENOENT). Normal: the session may not have written it yet.** |
| `error` | `OVERLAY_PLAN_ERROR` | `error reading plan` | `red` | inherited вЂ” genuine FS read failure (EACCES/non-ENOENT), light or GSD path alike |

**Locked literal:** `OVERLAY_PLAN_NO_LIGHT = 'session has not written a plan yet'` (CONTEXT D-04 suggestion,
confirmed). Exact wording remains Claude's Discretion within the D-04 bound, but the literal above **is the
recommended contract** and satisfies all three hard requirements:

1. **Lexically distinct** from `OVERLAY_PLAN_NO_PHASE` ("not a GSD sessionвЂ¦") and `OVERLAY_PLAN_NO_PLAN`
   ("phase has no PLAN.mdвЂ¦") and `OVERLAY_PLAN_ERROR` ("error reading plan") вЂ” no shared head phrase, no
   ambiguity. (Note: it shares the word "session" with no other plan copy and the word "plan" reads
   naturally for a quick session, where "PLAN.md" would be a GSD-specific lie.)
2. **NO_COLOR-legible** (inherited Phase 44 D-07): the operator distinguishes all four states by **words
   alone** with color stripped. "session has not written a plan yet" в‰  "phase has no PLAN.md yet" в‰ 
   "not a GSD session / no phase resolved" в‰  "error reading plan" as plain text.
3. **Honest & non-alarming**: describes the real, expected situation (a quick session that simply hasn't
   produced its lightweight plan) without implying a fault.

## Interaction & State Contract (terminal-TUI addendum вЂ” all INHERITED)

> The operator-visible interaction is **indistinguishable** from the Phase 44 GSD overlay. This phase
> changes the *reader's branching* inside `readPlan` (`plan.js`), not the *interaction*. Listed here as
> the interaction source of truth; every row is inherited verbatim from `44-UI-SPEC.md`.

| Concern | Contract | Status |
|---------|----------|--------|
| Open | `p` on the selected row (`filtered[sel.index]`); handler `p` (`App.js:482-502`) **does not change its shape** (D-01) | inherited |
| Snapshot | **Frozen** at open via `setOverlaySnapshot({ kind:'plan', taskRef, status, lines })`; live poll refreshes the table underneath, overlay content does not change until reopened | inherited |
| Scroll | `scrollOffset` over `lines[]`, slice `OVERLAY_VIEWPORT=18`; light-plan lines scroll identically to GSD lines | inherited |
| Close | `Esc` в†’ `mode:'list'`; cursor **preserved by `task_id`** | inherited |
| Async guard | `readPlan` is **synchronous** в†’ no `await` window в†’ the `p` open does **not** need an `overlayReqRef` capture (do not cargo-cult one). The light-plan `readFileFn` is sync too вЂ” divergence preserved | inherited (RESEARCH Pitfall) |
| Never-throws | ALL fs reads (incl. the new light-plan `readFile`) collapse to a discrete `status` (`ok`/`no-light-plan`/`error`/`no-phase`); no FS error reaches React. ENOENTв†’`no-light-plan`, otherв†’`error` (D-05/D-09) | inherited + extended to the fallback |
| Selection identity / anti-ReDoS | overlay opens over the `task_id`-revalidated row; the artifact path is **constructed** (`join(homedir(),'.kodo','plans',`${task_id}.md`)`), never regex-derived; optional `String.includes` containment guard on `task_id` (D-09), never `new RegExp` | inherited + applied to the new path |

**Status taxonomy after this phase (the discriminated union the render switches on):**
`ok` (render lines) В· `no-phase` (dim) В· `no-plan` (dim) В· **`no-light-plan` (dim, NEW)** В· `error` (red).
The fallback **narrows** when `no-phase` appears (now only when `task_id` is also falsy) but does not
remove it (D-06).
