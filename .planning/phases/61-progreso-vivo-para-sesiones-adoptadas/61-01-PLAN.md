# Phase 61 Plan 01: Progreso vivo para sesiones adoptadas

## Objetivo

Una sesión GSD **adoptada** (sin worktree de kodo) muestra su `N/M` en el dashboard igual que una lanzada. Se logra (a) relajando el gate del progreso de "flag `gsd`" a "STATE.md GSD legible en el path resuelto" (D-1, dinámico), (b) resolviendo el path con fallback worktree→project_path (D-2), y (c) marcando `gsd`/`gsd_mode`/`phase_id` al adoptar para las columnas phase/mode (D-3). Diseño locked en `61-CONTEXT.md`.

## must_haves (goal-backward)
1. Una sesión adoptada (sin worktree de kodo) con `<project_path>/.planning/STATE.md` poblado muestra `N/M` en la columna `prog`.
2. Una sesión LANZADA (con worktree) sigue mostrando `N/M` desde `.claude/worktrees/<sid>/.planning/STATE.md` (NO regresión).
3. El keep-last-good y `readGsdProgress` quedan intactos (solo cambian gate + resolución de path).
4. Una sesión sin STATE.md GSD en su path → `—` (sin cambio observable).
5. `adopt.js` setea `gsd:true` (+ `gsd_mode`/`phase_id` si derivables) cuando el cwd/project_path es proyecto GSD.
