## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator keystroke → state machine | `a`/`Esc` keystrokes drive the discover/picker/confirm machine; ink does not await async handlers |
| cmux host → dashboard | `listAgentSurfaces()` (in-process, typeof-gated) returns AgentSurface[] with string-validated fields (normalizeSurface) |
| dashboard → child process | onAdopt → runAdopt shells `kodo adopt` via execFile (no shell), invoked via process.execPath + absolute kodoBin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-05 | DoS (UX) | `a` async handler + onAdoptDiscover (App.js, index.js) | mitigate | onAdoptDiscover is typeof-gated and fails open to [] (host without the method → ADOPT_NONE footer); listAgentSurfaces is already never-throws (Phase 55); the handler never re-throws → ink panel stays mounted. |
| T-56-06 | Tampering | confirm-key routing (App.js Pitfall 2) | mitigate | The second key is routed by which armed-id is set (armedSessionId vs armedTaskId); an `a` cannot trigger a dismiss and a `d` cannot trigger an adopt. app-dismiss.test.js regression-guards the dismiss path. |
| T-56-07 | EoP | kodo binary resolution (index.js) | mitigate | kodoBin is an absolute join (no PATH lookup) and the spawn binary is process.execPath — mirror of the polling.js EoP mitigation. |
| T-56-08 | Information Disclosure | ADOPT_NO_PROJECT footer echoes cwd | accept | The footer shows the local cwd to the local operator on their own TTY; no remote disclosure surface. The escape-hatch message is intentional (D-05 fails loud toward the CLI). |
| T-56-09 | Spoofing/Tampering | no/ambiguous project → no shell | mitigate | resolveProjectId returning {error} blocks onAdopt entirely — a surface that cannot be unambiguously mapped to a project never shells `kodo adopt`, preventing adoption into the wrong project. |
| T-56-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — zero external dependencies (RESEARCH §Standard Stack). No legitimacy gate required. |
