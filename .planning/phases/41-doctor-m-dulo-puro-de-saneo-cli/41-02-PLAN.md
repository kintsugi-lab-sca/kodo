<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| doctor.execute → filesystem/git | execute deletes worktree dirs (via git), lock files, and NDJSON logs — destructive against the user's machine |
| scan snapshot → execute action | Time gap between detection and action: a dead resource can become live (PID reuse on macOS) |
| .bg-shell scope → other tools' worktrees | Mis-scoped detection could delete `.claude/worktrees/` or orca worktrees |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-04 | Tampering | execute() liveness re-check (TOCTOU / PID reuse) | mitigate | D-06/D-14: execute RE-detects + re-checks `isPidAlive`+`alive` IMMEDIATELY before each destructive action; never consumes scan()'s snapshot as a plan. Acceptance criteria require a test that flips liveness between scan and execute and asserts the target is skipped. Lock TTL is the secondary guard against PID-reuse (D-13). |
| T-41-05 | Denial of Service | worktree detection scope | mitigate | Detection enumerates `.bg-shell/<sessionId>` cross-checked against state.json ONLY — never `git worktree list`. Acceptance criterion: `grep -c "worktree list" == 0`. Guarantees `.claude/worktrees/` and orca worktrees are structurally invisible to doctor. |
| T-41-06 | Tampering | destructive primitives (no rm -rf) | mitigate | Worktree removal delegates to `cleanupWorktree` (git remove without --force, dirty→.dirty) / `git worktree prune`. Logs unlink WHOLE (never truncate). `grep -c "rm -rf" == 0` enforced. No raw `rmSync` of directories. |
| T-41-07 | Tampering | live-resource protection | mitigate | D-14: execute SKIPS any worktree/lock whose session is `alive===true` or PID alive. Test asserts a live-PID stub yields zero destructive calls. |
| T-41-08 | Information Disclosure | NDJSON log deletion | mitigate | Only non-live `.ndjson` past `DEFAULT_RETENTION_DAYS` cutoff are unlinked; live sessions' logs are never touched (D-12). Truncation forbidden (breaks POSIX followers). |
| T-41-SC | Tampering | npm/pip/cargo installs | accept | No package installs — composition over `node:fs` + existing internal modules only. |
</threat_model>
