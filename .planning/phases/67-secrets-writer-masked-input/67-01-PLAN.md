# Phase 67-01: writeEnvVar Module Foundation

**Status:** In planning  
**Wave:** 1 of 3  
**Date:** 2026-07-02  
**Depends on:** Phase 65/66 (atomic write patterns)  
**Blocks:** Phase 67-02 (masked input), Phase 67-03 (hygiene), Phase 68 (setup mode)

## Goal

Implement `writeEnvVar(key, value)` — the **only** writer of API key secrets to `~/.kodo/.env` with:
- Atomic writes (temp + chmod 0600 **pre-rename** + fs-atomic rename)
- Parse-merge-write (upsert single key, preserve other env vars)
- Secure permissions (0600, world-unreadable) on every write
- Prevents accidental clobbering of other secrets

## Scope: In

- `writeEnvVar(key, value)` function in `src/config.js`
- Atomic write sequence: `writeFileSync(tmp)` → `chmodSync(tmp, 0o600)` → `renameSync(tmp, ENV_PATH)`
- Parse-merge logic (read existing `.env`, upsert key, preserve rest, write back)
- Format: `KEY=VALUE\n` (no quotes, no escaping)
- Validation: reject keys/values with `#`, embedded `=`, or leading spaces
- Integration: export from `src/config.js`

## Scope: Out

- Setup mode / first-run logic (Phase 68)
- Masked input UI (Phase 67-02)
- Hygiene grep tests (Phase 67-03)
- Hot-reload (CFGF-01, v2)

## Success Criteria

1. ✅ `writeEnvVar('PLANE_API_KEY', 'key123')` creates `~/.kodo/.env` with `0600` permissions
2. ✅ `writeEnvVar('GITHUB_TOKEN', 'token456')` called after plane key preserves `PLANE_API_KEY` untouched
3. ✅ No intermediate `.env.tmp` files left behind (atomic rename)
4. ✅ `ls -l ~/.kodo/.env` = `-rw-------` (perms 0600) immediately after write
5. ✅ Graceful error handling if dir creation fails (never-throws pattern)
6. ✅ Unit tests exercise merge logic, permissions, atomic safety
