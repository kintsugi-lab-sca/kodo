## Goal

Extend the Phase 63 text-input component to support **masked rendering** (display `‚ÄĘ` per character) for entering the API key, preserving the value in memory but never rendering it raw. Integrate with `writeEnvVar` via DI callback (`onSaveApiKey`).

## Scope: In

- Mask flag (`mask` prop) on text-input component in `src/cli/dashboard/App.js`
- Render logic: if `mask=true`, paint `‚ÄĘ` instead of actual character
- `onSaveApiKey` DI callback ‚Üí calls `writeEnvVar(key, value)` (never shell-out)
- `[configurado]` indicator: show presence, never the value
- Graceful degradation: non-TTY or no raw-mode support ‚Üí fall back to `kodo config` wizard (never-throws)

## Scope: Out

- Setup mode UI or first-run flow (Phase 68)
- Hygiene grep or value leakage testing (Phase 67-03)
- Reveal/toggle button for unmasking (deferred, Pitfall 11)
- Paste support for masked fields (deferred, nice-to-have)

1. ‚úÖ Text input in `config-edit` mode with `mask=true` displays `‚ÄĘ‚ÄĘ‚ÄĘ‚ÄĘ‚ÄĘ` instead of raw value
