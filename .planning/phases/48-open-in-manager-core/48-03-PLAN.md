<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Open-in-manager: pressing `o` on a dashboard row opens the task in the system browser via
    `execFile('open', [url])` fire-and-forget (never-throws, http(s)-allowlisted), with a green
    transient `opening <ref>…` footer; a bare `no task URL for this session` footer on legacy
    rows; and the Plane browse-URL routed through the new optional `plane.web_url` config (split
    web/API deploy fix), with `UNKNOWN-<seq>` treated as "no URL".
  </what-built>
  <how-to-verify>
    Run these on your macOS machine with the kodo server running and at least one session in the
    dashboard. Start: `kodo dashboard`.

    1. HAPPY PATH (OPEN-01): Move the cursor (↑/↓) to a row that has a task (Plane or GitHub) and
       press `o`. EXPECT: your default browser opens a tab on the LIVE web UI of that task; the
       dashboard footer briefly shows a green `opening PROJ-123…` (using the row's ref). The
       dashboard panel stays exactly where it was — no flicker, no header stacking, cursor still
       on the same row.

    2. ALT-SCREEN SURVIVAL (OPEN-02 / Pitfall 3): After step 1, press any key (e.g. ↓). EXPECT:
       the green footer clears and the hints line (`… · d dismiss · o open · q quit`) returns;
       the terminal is NOT garbled; on `q` the original scrollback is restored cleanly.

    3. LEGACY NO-OP (OPEN-02 / SC#2): If you have a session launched before v0.12 (no task_url)
       — or temporarily edit a row's record to clear task_url — press `o` on it. EXPECT: NO tab
       opens; the footer shows the bare `no task URL for this session` (no `[!]`, no
       `— press any key`). Pressing any key clears it.

    4. ADVERSARIAL REFUSAL (OPEN-03): (Optional, if you can craft a row with a non-http(s)
       task_url such as `file:///etc/passwd` or a leading-dash value.) Press `o`. EXPECT: NO tab,
       NO Calculator/app launch; footer shows `[!] refused non-http(s) URL — press any key`.

    5. SPLIT-DEPLOY PLANE URL (OPEN-04 / SC#5): In `~/.kodo/config.json`, set
       `providers.plane.web_url` to your Plane WEB host (different from `base_url` if you run a
       split deploy; or temporarily set it to a distinct test URL). Restart the dashboard, press
       `o` on a Plane row. EXPECT: the opened tab uses the `web_url` host, NOT the API host.
       Confirm a row whose identifier is `UNKNOWN-…` shows the no-URL footer rather than opening a
       dead link.
  </how-to-verify>
