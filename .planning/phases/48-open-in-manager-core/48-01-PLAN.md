<objective>
Fix the latent Plane browse-URL bug (OPEN-04): the URL is built from the API host
(`base_url`), which is dead on split web/API self-hosted deploys. Route URL construction
through an optional `plane.web_url` config (default = `base_url`), wired end-to-end from
config → registry factory → provider context builders → normalizer. Additionally, suppress
the URL entirely when the project identifier is unresolved (`UNKNOWN-<seq>`) so no dead link
is ever emitted.

Purpose: This is the data-layer half of Phase 48. It is the source-of-truth correctness fix
that makes the `task_url` persisted on every SessionRecord a *live* web link in all deploy
topologies. It must be wired through the whole call chain — a one-line `normalize.js:76` edit
is inert unless the caller passes `webUrl` from config (CONTEXT caveat / D-07).

Output: `plane.web_url` config field (resolve-on-read default), end-to-end webUrl threading,
UNKNOWN-suppression, and a normalizer test matrix covering unified deploy / split deploy /
UNKNOWN.
</objective>

## Trust Boundaries

|----------|-------------|
| Plane API response → normalizeWorkItem | Provider-controlled fields (`sequence_id`, `project_detail.identifier`) cross into the URL string that later reaches the launcher. |
| Operator config (`~/.kodo/config.json`) → registry factory | `plane.web_url` is operator-supplied and becomes the URL host. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-01 | Information Disclosure / Tampering | normalize.js url field | mitigate | UNKNOWN/falsy identifier suppresses the url (no dead/wrong link emitted into the persisted SessionRecord); ref keeps human slug + sequence_id, never the workspace/issue UUID. |
| T-48-02 | Spoofing | plane.web_url config | accept | web_url is operator-controlled local config (same trust level as base_url); a malicious local config is out of threat scope. The launcher (Plan 48-02) still enforces an http(s) allowlist on whatever URL is produced — defense in depth. |
| T-48-03 | Tampering | Plane response feeding the url | mitigate (downstream) | Any crafted URL value produced here is re-validated by the http(s) protocol allowlist in open.js (Plan 48-02, OPEN-03) before reaching execFile — this plan does not itself launch anything. |
| T-48-SC | Tampering | npm/pip/cargo installs | n/a | No package installs in this plan (Node built-ins + existing deps only). |
