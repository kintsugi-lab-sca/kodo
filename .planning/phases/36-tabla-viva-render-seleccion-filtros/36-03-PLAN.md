<objective>
Wire keyboard interaction onto the table from Plan 02: mode-gated `useInput` that routes keys through a `mode: 'list' | 'filter'` flag. In list mode, ↑/↓ move the derived cursor (clamp, no wrap) re-fixing `selectedTaskId`, and `/` enters filter mode. In filter mode (modal), printable chars append to `query` (live re-filter), Backspace pops, Enter confirms (keeps filter), Esc cancels (clears query) — the D-15 modal-scope Esc, distinct from the list-mode Esc reserved for Phase 38. The live `query` now feeds `parseFilter`/`applyFilter` in the render pipeline, and `SessionTable` gains the filter line + the `no sessions match` empty branch.

Purpose: TUI-08 (cursor navigation tracked by identity) and TUI-12 (modal filter with cursor preservation) — the two interactive requirements. Both load-bearing invariants already have PURE coverage from Plan 01 (`resolveSelection`); this plan delivers the interactive surface and its render-level tests.

Output: extended `App.js` (key routing + query/mode state), extended `SessionTable.js` (filter line + no-match state), and added interaction tests in `test/dashboard-table.test.js`.
</objective>
