**Imports:**
```javascript
// @ts-check
import { readFileSync, writeFileSync, unlinkSync, existsSync, mkdirSync, realpathSync } from 'node:fs';
import { join, dirname } from 'node:path';
```

**Constants:**
- `LOCK_FILE = '.planning/.kodo.lock'` (per D-05)
- `DEFAULT_TTL_HOURS = 4` (per D-06)

**JSDoc typedef `LockContent`** with fields per D-06:
- `session_id: string`
- `task_id: string`
- `task_ref: string`
- `pid: number`
- `acquired_at: string` (ISO-8601)
- `ttl_hours: number`

**Exported functions:**

1. `isPidAlive(pid)` — uses `process.kill(pid, 0)`, returns `true` if alive. Catches error: returns `false` only when `e.code === 'ESRCH'` (no such process). Other errors (e.g. EPERM) mean PID exists but we lack permission, so return `true`.

2. `readLock(projectPath)` — resolves path via `realpathSync(projectPath)`, reads `join(resolved, LOCK_FILE)`. Returns parsed JSON or `null` if file absent or corrupt (try/catch parse).

3. `acquireGsdLock(projectPath, sessionInfo)` — per D-07 four-case semantics:
   - Case 1: Lock file absent -> `mkdirSync(dirname(lockPath), { recursive: true })`, write lock JSON, return `{ acquired: true }`.
   - Case 2: Lock exists, `isPidAlive(existing.pid)` is false -> steal (overwrite with new lock), return `{ acquired: true }`.
   - Case 3: Lock exists, PID alive, TTL expired (`Date.now() - new Date(existing.acquired_at).getTime() > existing.ttl_hours * 3600_000`) -> steal + `console.error('[kodo:lock] Stealing expired lock from ${existing.task_ref} (acquired ${existing.acquired_at}, TTL ${existing.ttl_hours}h exceeded)')`, return `{ acquired: true }`.
   - Case 4: Lock exists, PID alive, TTL valid -> return `{ acquired: false, holder: existing }`.
   - Corrupt lock file (JSON parse fails) -> treat as stale, steal.

4. `releaseGsdLock(projectPath, sessionId)` — per D-09 idempotent release:
   - If lock file absent -> return silently (no-op).
   - If lock file exists, read and parse. If `existing.session_id === sessionId` -> `unlinkSync(lockPath)`. If session_id mismatch -> leave lock (another session owns it). If JSON parse fails -> delete (corrupt).

**Private helper `stealLock(lockPath, sessionInfo, reason)`:** writes new lock content, returns `{ acquired: true }`. Logs reason to stderr: `console.error('[kodo:lock] Lock stolen: ${reason}')`.

**Key constraints:**
- Do NOT import from `../config.js` — lock path comes from `projectPath` argument (per PATTERNS.md).
- Always use `realpathSync(projectPath)` before building lock path (Pitfall 3: symlink divergence on macOS).
- `mkdirSync(dirname(lockPath), { recursive: true })` before every write (Pitfall 4: `.planning/` may not exist).
