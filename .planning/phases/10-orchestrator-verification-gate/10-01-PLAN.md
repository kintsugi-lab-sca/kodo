## Trust Boundaries

| Boundary | Description |
|----------|-------------|

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Tampering | parseVerificationFrontmatter | mitigate | Fail-closed: nunca lanzar excepción; cualquier malformación retorna `{ error }`. Input no-string → `{ error }` sin acceder a propiedades. |
| T-10-01-02 | DoS | parseVerificationFrontmatter | accept | Un VERIFICATION.md de 1GB podría colgar el regex. Accept: el archivo lo escribe Claude/humano local, no hay vector remoto. Si se volviese issue, añadir límite `md.length < 1MB`. |
| T-10-01-03 | Tampering | parseVerificationFrontmatter | mitigate | YAML injection sin parser YAML real es N/A — el regex hand-rolled extrae SOLO 4 campos escalares conocidos; cualquier constructo YAML exótico (anchors, aliases, multilinea, tags) se ignora silenciosamente. Documentar en JSDoc que "extra fields and YAML features are ignored by design". |
| T-10-01-04 | Info Disclosure | computeVerdict | accept | El `detail` puede incluir el status string del archivo. No contiene secretos (status es enum cerrado + counts numéricos). Accept. |
| T-10-01-05 | Elevation | parseVerificationFrontmatter | mitigate | Prototype pollution vía `__proto__: ...` como key del frontmatter. El regex `/^(\w+):...` con `\w+` NO matchea `__proto__` (no incluye caracteres especiales; `\w` = `[a-zA-Z0-9_]`, y `__proto__` SÍ matchea `\w+`). Mitigar usando `Object.create(null)` o `Reflect.set` para el objeto out; alternativa: check `if (key === '__proto__' \|\| key === 'constructor') continue;`. |
</threat_model>
