## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-02-01 | Denial of Service | cleanupFn que cuelga indefinidamente | accept | Caller (bin/kodo) controla deps; en producción no se inyecta cleanupFn. Tests usan callbacks bounded. Sin escalación cross-trust-boundary. |
| T-31-02-02 | Tampering | cleanupFn throws → return code mutado | mitigate | Patrón try/finally estándar Node: si finally throws, la excepción reemplaza el return — comportamiento documentado. Tests no exercise este path; bin/kodo no inyecta cleanupFn, así que el surface efectivo es 0 en producción. |
| T-31-02-03 | Repudiation | Cleanup completa después de exit (sin la fix) | mitigate | Esta es exactamente la advisory que ADVISORY-02 cierra. Los 3 tests `process.hrtime.bigint()` blindan el ordering observable. |
| T-31-02-04 | Information disclosure | cleanupFn ve opts/deps con paths de filesystem | accept | Caller-controlled scope; no input externo cruzando boundary. |

ASVS L1: este plan no introduce input externo ni surface de red. El DI hook es interno (module-level), invocado solo por consumers in-process.
