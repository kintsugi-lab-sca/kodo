## DECISIÓN ARQUITECTÓNICA — Opción A (Lifecycle Simulator Hook)

CONTEXT.md D-09/D-10/D-14 asume que `launchOrchestrator` hace `child_process.spawn`. **Realidad verificada**: `launchOrchestrator` solo invoca `cmux.*` (newWorkspace, listWorkspaces, send, setColor, notify) — NO hay `child_process.spawn`/`exec`/`fork` en `src/orchestrator/launch.js`. El lifecycle real lo hace el binario `claude` que cmux arranca dentro del workspace tras `cmux.send`.

**Adoptada por este plan:** **Opción A — "Lifecycle simulator hook"** (recomendada por el orquestador en critical_planning_input).

`opts.spawnFn` se interpreta como un hook DI OPCIONAL que `launchOrchestrator` invoca **DESPUÉS** de `cmux.send` y **ANTES** del `return { workspace: workspaceRef, existing: false }`. Recibe un objeto con `{ workspaceRef, sessionId, projectPath, kodoDir, taskRef }`. En producción (callers existentes), `spawnFn` es `undefined` → el `if (opts.spawnFn)` lo elide silenciosamente: el lifecycle real lo hace claude binary dentro del workspace cmux (vía la canónica `cmux.send`). En tests, `spawnFn` es inyectado y ejecuta el `node -e <inline-script>` que simula `addSession` + NDJSON `session.start` con `transcript_path` populated.

**Justificación de la elección:**
1. Preserva la intención de SC#3 (validar observables reales post-lifecycle).
2. Mantiene el patrón DI ya establecido en Phase 31 (`onConsoleWarn` en 31-01, `cleanupFn` en 31-02 — todos NULL+if-guard).
3. Default `undefined` (no `child_process.spawn` real default) preserva back-compat byte-exact: el bundle de prod no carga `child_process` desde `launch.js`.
4. Produce un test que valida que `launchOrchestrator` engancha correctamente al sistema de sesiones — verifica el contrato observable, no la fidelidad del runtime cmux.
5. NO requiere refactor de cmux client ni mock de proceso real claude.

**Opción B descartada** (test fixture externo que ejecuta spawn fuera del SUT): más limpio para `src/launch.js` pero artificial en test — no valida que `launchOrchestrator` engancha al lifecycle.

**Opción C descartada** (mover ADVISORY-03 a v0.9+ backlog): violaría ROADMAP SC#4 (deferred section → 0 items).

Real spawn helper:
```
import { spawn as realSpawn } from 'node:child_process';
async function runInlineNode(script, env) {
  return new Promise((resolve, reject) => {
    const child = realSpawn(process.execPath, ['-e', script], { env, stdio: ['ignore', 'pipe', 'pipe'] });
    let stdout = '', stderr = '';
    child.stdout.on('data', d => stdout += d);
    child.stderr.on('data', d => stderr += d);
    child.on('error', reject);
    child.on('exit', code => resolve({ code, stdout, stderr }));
  });
}
```
