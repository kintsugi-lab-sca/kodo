    - acquireLock(freePath, {retries:0}) → returns { token } (a randomUUID); lockfile now exists with JSON {pid, acquired_at, token}.
    - acquireLock(heldPath, {retries:0}) while held by a live foreign owner → returns null.
    - acquireLock steals when the stored pid is dead (isPidAlive false) OR (now - acquired_at) > ttlMs; steal write is tmp+rename.
    - acquireLock with retries>0 backs off (short, ~20ms) between attempts and succeeds once the holder releases within the window.
    - releaseLock(path, token) unlinks ONLY if the stored token matches; wrong/absent token = no-op; corrupt lock = removed.
    - withFileLock(path, fn, opts): acquires; on success returns { ok:true, value: fn() } and releases in finally; on acquire failure returns { ok:false, reason:'lock-timeout' } + a warn, never throws.
  </behavior>
  <action>
    Implement the primitive as a small pure module (~40-60 lines). Import `isPidAlive` from '../gsd/lock.js' (D-01: reuse, never reimplement).

    Export three functions:
    - `acquireLock(lockPath, { retries = 8, backoffMs = 20, ttlMs = 10_000 } = {})`: generate `token = randomUUID()`; content = JSON.stringify({ pid: process.pid, acquired_at: Date.now(), token }). Loop i in 0..retries: try `writeFileSync(lockPath, content, { flag: 'wx' })` → return { token }. On EEXIST, read the existing lock; if JSON.parse fails treat as stale; compute `stale = !isPidAlive(held.pid) || (Date.now() - held.acquired_at > ttlMs)`; if stale, steal via tmp+rename (`lockPath + '.steal.' + process.pid + '.' + randomUUID()`, writeFileSync then renameSync) and return { token }; else (live + within TTL) sleep backoffMs and retry. Any non-EEXIST fs error rethrows. Return null after exhausting retries. Ensure the parent dir exists (mkdirSync recursive) before first write.
    - `releaseLock(lockPath, token)`: ownership-checked, idempotent — read+parse; if held.token === token unlinkSync; corrupt → unlinkSync; missing/other-owner → no-op; wrap in try/catch so it never throws.
    - `withFileLock(lockPath, fn, opts = {})`: `const got = acquireLock(lockPath, opts)`; if !got, emit an NDJSON warn (use the project logger if injectable via opts.logger, else a console.warn fallback keyed 'lock.timeout' with { lockPath }) and return `{ ok: false, reason: 'lock-timeout' }`; else try `return { ok: true, value: fn() }` finally `releaseLock(lockPath, got.token)`.

    Use short backoff via a synchronous-safe sleep is NOT possible for sync fn; keep the retry loop synchronous using a tight busy-wait bounded by backoffMs*retries (acceptable — state writes are sub-ms and retries are few), OR block via Atomics.wait on a throwaway Int32Array for backoffMs. Prefer Atomics.wait(new Int32Array(new SharedArrayBuffer(4)), 0, 0, backoffMs) so the retry sleeps without spinning the CPU and keeps withFileLock fully synchronous (matches the sync state mutators).
