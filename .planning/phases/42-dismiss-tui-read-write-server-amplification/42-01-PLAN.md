<objective>
Amplify the existing `DELETE /sessions/{id}` HTTP handler from a 3-line static `removeSession(taskId)` no-op into a defense-in-depth destructive-mutation endpoint that delegates real sanitization to `doctor.execute` (Phase 41) through a pure, dependency-injected, unit-testable module — mirroring the Phase 40 `src/server/provider-state.js` extraction precedent.

This plan delivers the **server half** of Phase 42 (DISMISS-01 + the server+doctor layers of DISMISS-04). The TUI half (Plan 02) consumes the HTTP contract this plan defines but shares zero files with it.

Purpose: A dismiss path that omits `fix:true` reports success while sanitizing nothing (RESEARCH DRIFT #2 — `doctor.js:468`); and the current handler is structurally untestable (inline closure, static import, zero tests — DRIFT #3). This plan makes the destructive path testable and correct, and is the authoritative TOCTOU re-check (the 409 guard is SC#3).

Output: `src/server/dismiss.js` (new pure module), an amplified thin DELETE adapter in `src/server.js`, a `SESSION_DISMISSED` aggregate event, and `test/server/dismiss.test.js` covering the 409 guard, the counter→actions translation, the `fix:true` lock, and never-throws.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI/web client → DELETE /sessions/{id} | Untrusted client request crosses into a destructive server mutation. The server must NOT trust the client's claim that the target is dead. |
| server → doctor.execute (filesystem + git worktree + state.json) | The mutation actually deletes worktrees, steals locks, removes state entries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-01 | Tampering | DELETE handler dismissing a LIVE session (race or malicious/buggy client that bypassed the TUI guard) | mitigate | Server-side 409 guard in `dismiss.js`: re-read fresh `loadState().sessions[taskId]`; if `alive===true` return 409 and NEVER call executeFn. Authoritative layer (D-07/D-08). Doctor's per-action liveness re-check is the final net. (V4 Access Control) |
| T-42-02 | Tampering / TOCTOU | session revives between the client's snapshot and the DELETE arriving | mitigate | The 409 guard reads state at DELETE-receive time, not from a client snapshot (SC#3). Tested deterministically with a `loadState` that flips `alive` before the call. |
| T-42-03 | Repudiation | dismiss reports `{ok:true}` while sanitizing nothing (fix falsy → silent no-op, DRIFT #2) | mitigate | `fix:true` locked in `dismiss.js` + unit test asserting the executeFn spy received `{fix:true}`; `SESSION_DISMISSED` NDJSON event makes the mutation auditable. |
| T-42-04 | Tampering | path traversal via `task_id` in `/sessions/<id>` | mitigate | `decodeURIComponent` retained (server.js:496); doctor scopes to a single session's worktree/lock/state and never does recursive `rmSync` (Phase 41 register). (V5 Input Validation) |
| T-42-05 | Tampering (data loss) | worktree with uncommitted work deleted | mitigate | doctor moves dirty worktrees to `.dirty` instead of deleting (Phase 41 D-11); translated to `moved-dirty` in actions[] so Plan 02 surfaces it. |
| T-42-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs this phase (RESEARCH §Package Legitimacy Audit: N/A). No [ASSUMED]/[SUS]/[SLOP] packages. |
</threat_model>
