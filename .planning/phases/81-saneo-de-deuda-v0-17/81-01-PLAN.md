<objective>
Cerrar DEBT-01: la semántica de clear/stale de `next` en `upsertTaskHandoff`. Hoy el merge (`state.js`) conflacia `null` y `undefined` vía `??` (ambos preservan) y el único caller (`session-end.js`) embudo AMBAS ramas de autoría (LLM y backstop mecánico) al mismo `next: null` — así nada puede borrar nunca un `next` obsoleto. Este plan implementa el contrato de tres estados (D-01): string no-vacío → sobrescribe; `null` explícito → clear deliberado; campo ausente → preserva; y mapea la autoría al contrato en el caller (D-02): LLM sin `NEXT:` → `null` (clear); backstop mecánico → omite el campo (preserve).

Purpose: un cierre de sesión sin `NEXT:` redactado por el LLM deja de dejar un puntero `next` obsoleto para siempre, SIN regresionar la asimetría de Phase 74 (74/WR-02: un cierre mecánico no debe ni resucitar ni borrar un `NEXT:` real) ni el nudge LIVE-07 de Phase 75.

Output: writer de tres estados + caller discriminado + tests reexpresados y ampliados.
</objective>
