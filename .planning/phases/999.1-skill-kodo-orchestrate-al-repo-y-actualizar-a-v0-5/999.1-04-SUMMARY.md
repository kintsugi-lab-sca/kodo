## Decisions Made

- **spawnSync child en lugar de dynamic import** — `handleOrchestratorStop` no está exportada y la firma cero-args (H5) prohíbe importarla. El child resuelve `KODO_ROOT` desde `process.env` al import-time, lo que hace innecesario importar el módulo: basta con `spawnSync(node, [STOP_HOOK], { env: { KODO_ROOT, HOME }, input: JSON.stringify({...}) })`.
- **Doble tmpdir aislado (HOME + repo)** — separar `tmpHome` (para `~/.kodo/state.json` y `~/.kodo/locks/`) de `tmpRepo` (workdir git del hook) clarifica la intención del test. Aunque el path `handleOrchestratorStop` no toca state.json (porque `findSession` retorna `null` cuando no hay state.json sembrado), la separación previene side-effects si el flujo del hook añade más I/O al ~/.kodo en el futuro.
- **`commit.gpgsign false` local** — el plan exige no usar `--global`; aplicarlo local al tmpRepo desactiva firmas GPG sin tocar el gitconfig del desarrollador. Mitiga T-999.1.04-04 (CI con `commit.gpgsign=true` global se colgaría esperando passphrase).
- **`timeout: 10000` en `spawnSync`** — mitigación T-999.1.04-03 (DoS): un hook colgado nunca debe colgar la suite. 10s es generoso para un proceso que solo hace `git status` + `git commit` en tmpdir.
- **Test NO importa el módulo del hook** — ni `handleOrchestratorStop` ni `runStopHook`. Honra H5 al 100%: el plan declara "src/hooks/stop.js no se modifica", y no importar el módulo hace que cualquier cambio futuro en la API interna del hook no rompa este test (solo el contrato CLI público).
